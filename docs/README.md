# HappyMeasure Documentation

Start with these public docs:

- `HARDWARE_VALIDATION_PROTOCOL.md` for bench validation order.
- `HARDWARE_DRY_RUN_GUIDE.md` for no-DUT and simulator checks.
- `WINDOWS_PORTABLE_BUILD.md` for building a portable Windows folder app.
- `TESTING_STRATEGY.md` for software validation commands.
- `TROUBLESHOOTING.md` for common runtime and hardware issues.
- `CHANGELOG.md` for release history.
- `../CONTRIBUTING.md` for the documentation rule used on every change.

Internal handoff notes, temporary agent context, legacy migration archives, and
local runtime logs are intentionally not included in the public release tree.
